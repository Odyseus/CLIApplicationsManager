Movies DB's documentation

A CLI utility written in Python 3 to create a movies data base on GNU/Linux.

Hint

This Python application can generate a system executable with Bash completions support. See app.py generate.

CLI Synopsis

Movies DB 2021-09-27 08:34:09.467 (Unstable)

CLI utility to create a movies data base on GNU/Linux.

Usage:
    app.py (-h | --help | --manual | --version)
    app.py movies (scan | base_data | detailed_data) [--debug]
    app.py server (start | stop | restart)
                  [--host=<host>]
                  [--port=<port>]
    app.py generate system_executable

Options:

-h, --help
    Show this screen.

--manual
    Show this application manual page.

--version
    Show application version.

--host=<host>
    Host name. [Default: 0.0.0.0]

--port=<port>
    Port number. [Default: 8889]

--debug
    Debug.

Sub-commands for the `movies` command:
    scan                                Scan directories for movies.
    base_data                           Generate base movies data.
    detailed_data                       Generate detailed movies data.

Sub-commands for the `server` command:
    start                               Start server.
    stop                                Stop server.
    restart                             Restart server.

Sub-commands for the `generate` command:
    system_executable    Create an executable for this application on the system
                         PATH to be able to run it from anywhere.

Requirements

Warning

  • Python 3.5+.

  • guessit>2 Python module.

  • requests Python module.

Detailed usage

CLI options

  • -h or --help: Display this application basic help screen.

  • --manual: Display this application manual.

  • --version: Show this application version.

Commands

app.py server

The HTTP server started will host the folder at UserData/www.

Sub-commands

  • start: Start HTTP server.

  • stop: Stop HTTP server.

  • restart: Restart HTTP server.

Options

  • --host=<host> Host name.

  • --port=<port> Port number.

app.py generate

Sub-commands

  • system_executable: Create an executable for the app.py application on the system PATH to be able to run it from anywhere.

    • The system executable creation process will ask for an executable name and the absolute path to store the executable file (the default is $HOME/.local/bin).

    • It will also ask for bash completions creation.
